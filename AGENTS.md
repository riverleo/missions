# Agent Guidelines

## 핵심 게이트
- 모든 코드 작성 및 파일 변경은 플랜(`plans/*.md`)을 기반으로만 작업한다.
- 작업 중 사용자의 새 요구사항이 있을 경우 기본 동작은 플랜에 기록이고 구현은 하지 않는다.

## 역할별 상세 규칙
- `플래너` (`docs/agents/planner/AGENTS.md`)
  - 플랜 작성/정리만 수행하며 코드 작성은 금지한다.
- `게임 디자이너` (`docs/agents/game_designer/AGENTS.md`)
  - 플랜에 정의된 게임 로직(유저 인터페이스, 게임 규칙, 게임 데이터) 구현을 수행한다.
- `플랫폼 엔지니어` (`docs/agents/platform_engineer/AGENTS.md`)
  - 플랜에 정의된 플랫폼 로직(백엔드 연동, 데이터/타입, UI/인프라/도구 체인) 구현을 수행한다.
- `테스트 엔지니어` (`docs/agents/qa_engineer/AGENTS.md`)
  - 플랜에 정의된 테스트 코드(단위 테스트, 명세-테스트 매핑, 회귀 검증) 구현/검증을 수행한다.

## 역할 호출 템플릿
- 특정 역할로 작업 요청 시 아래 형식을 사용한다.
	`[역할명]`
	`요청 내용`
- 복수 역할 동시 호출은 아래 형식을 사용한다.
	`[역할1, 역할2]`
	`<플랜파일명(확장자 제외)> 플랜 구현 시작`
- 예시:
	`[게임 디자이너, 테스트 엔지니어]`
	`20260220175103_plan_action_process_base 플랜 구현 시작`

## 문서 운영 원칙
- 루트 `AGENTS.md`는 게이트 규칙과 인덱스만 유지한다.
- 루트 `AGENTS.md` 수정 권한은 플래너에게만 있다.
- 새 규칙 추가 시 루트에는 요약만 남기고 상세는 역할 문서에 기록한다.
- 동일 규칙의 중복 기록을 피하고 단일 출처를 유지한다.
