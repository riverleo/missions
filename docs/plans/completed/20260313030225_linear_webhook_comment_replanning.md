# Linear 웹훅 코멘트 형식 및 재플래닝 개선

## 목표

Linear 웹훅 자동화가 남기는 이슈 코멘트를 역할 중심의 일관된 형식으로 통일하고, 사용자가 이슈 코멘트로 수정 요청을 남겼을 때 플래너가 이를 읽어 기존 플랜을 갱신한 뒤 구현 역할을 다시 지시할 수 있도록 흐름을 확장한다.

이번 플랜은 기존 상태 기반 자동화 위에 코멘트 규약과 재플래닝 분기를 추가하는 범위로 한정하며, 자동화 중복 실행 방지와 코멘트 루프 방지도 함께 검증 가능해야 한다.

## 담당자

- 플래너
- 플랫폼 엔지니어
- 테스트 엔지니어

## 할 일

### 플래너

- [x] 자동화 코멘트 형식을 `[역할] 진행사항`으로 통일하는 규칙과 적용 대상을 플랜 노트 및 구현 입력 기준으로 확정한다.
- [x] 사용자 작성 코멘트를 수정 요청으로 해석하는 조건과 플래너 재실행 범위를 정의한다.
- [x] 기존 플랜 갱신 시 새 플랜 생성 여부, 기존 플랜 수정 방식, 구현 역할 재지시 방식 중 저장소 운영 규칙에 맞는 절차를 문서화한다.

### 플랫폼 엔지니어

- [x] Linear 코멘트 발행 로직이 모든 자동화 단계에서 `[역할] 진행사항` 형식으로 메시지를 남기도록 수정한다.
- [x] 자동화가 작성한 코멘트와 사용자가 작성한 코멘트를 구분해 사용자 코멘트만 재플래닝 후보로 처리하도록 웹훅 분기 로직을 확장한다.
- [x] 사용자 코멘트가 들어오면 최신 플랜 문서와 코멘트 내용을 플래너 입력에 포함해 플랜 갱신 세션을 실행하도록 구현한다.
- [x] 플랜 갱신 후 구현 역할 재지시가 기존 worktree, PR, 상태 전이와 충돌하지 않도록 중복 실행 방지 조건을 보강한다.
- [x] 자동화가 남긴 코멘트에 다시 반응해 재플래닝이 반복되는 루프를 막는 보호 조건을 추가한다.

### 테스트 엔지니어

- [ ] 자동화 단계별 코멘트가 `[역할] 진행사항` 형식으로 생성되는지 실제 Linear 웹훅 흐름 기준으로 검증한다.
- [ ] 사용자가 이슈 코멘트로 수정 요청을 남겼을 때 플래너가 플랜을 갱신하고 구현 역할 재지시까지 이어지는지 검증한다.
- [ ] 자동화 코멘트, 봇 코멘트, 사용자 코멘트가 서로 다른 분기로 처리되어 재플래닝 루프가 발생하지 않는지 검증한다.

## 노트

### 2026-03-13

- 사용자 신규 요구사항 1: 자동화가 남기는 Linear 이슈 코멘트는 `[역할] 진행사항` 형식을 사용해야 한다. 예: `[테스트 엔지니어] 테스트 작성을 시작했습니다.`
- 사용자 신규 요구사항 2: 사용자가 Linear 이슈에 코멘트를 남기면 플래너가 해당 코멘트를 읽고 플랜을 수정한 뒤 구현 역할을 다시 지시해야 한다.
- 기존 `Linear 웹훅 로컬 서버 추가` 플랜의 후속 확장 작업으로 분리한다.
- 구현 시 사용자 코멘트와 자동화 코멘트의 구분 기준, 재플래닝 실행 조건, 기존 PR 및 worktree 재사용 정책을 명확히 해야 한다.
- 구현: `scripts/linear-webhook-workflow.js`에 자동화 코멘트 숨김 마커, `[역할] 진행사항` 포맷, 사용자 코멘트 기반 재플래닝 프롬프트, 대기 중 재플래닝 큐를 추가했다.
- 수정: Linear 코멘트 본문에 노출되던 HTML 주석 마커를 제거하고, 자동화 코멘트 판별은 역할 프리픽스 패턴 기준으로 변경했다.
- 검증: `src/test/linear-webhook-workflow.spec.js`에서 코멘트 포맷, 자동화 코멘트 제외, 사용자 코멘트 감지, 재플래닝 프롬프트 생성을 단위 테스트로 검증했다.
- 미완료 검증: 실제 Linear 이슈 코멘트 생성/수정 이벤트를 통한 종단 간 재플래닝 재현은 아직 수행하지 않았다.
