# PLAN: 새 아이템 생성 시 기본 상호작용 액션 자동 추가

## 목표
건물/캐릭터/아이템 상호작용에서 새 아이템이 생성될 때, 기본 상호작용 액션이 자동으로 생성되도록 한다.

## 범위
- 포함: 아이템 생성 시점의 기본 상호작용 액션 자동 생성 규칙
- 포함: 최초 생성 액션의 `root` 플래그 규칙
- 제외: 기존 상호작용 액션 일괄 마이그레이션
- 제외: UI/어드민 편집 화면 개편

## 명세
- [x] 건물 상호작용에서 새 아이템 생성 시 기본 상호작용 액션이 자동으로 추가된다.
- [x] 캐릭터 상호작용에서 새 아이템 생성 시 기본 상호작용 액션이 자동으로 추가된다.
- [x] 아이템 상호작용에서 새 아이템 생성 시 기본 상호작용 액션이 자동으로 추가된다.
- [x] 해당 아이템에서 최초로 생성되는 상호작용 액션은 `root = true`로 생성된다.
- [x] 해당 아이템에서 2번째 이후로 생성되는 상호작용 액션은 `root = false`로 생성된다.
- [x] 해당 아이템의 상호작용 액션이 모두 삭제된 상태라면, 이후 새로 추가되는 첫 액션은 다시 `root = true`로 생성된다.

## 수용 기준
- [x] 새 아이템 생성 직후 상호작용 액션 목록이 비어 있지 않다.
- [x] 최초 액션의 `root` 값이 항상 `true`다.
- [x] 동일 아이템에 액션을 추가 생성해도 `root = true` 액션은 1개만 유지된다.
- [x] 기존 액션을 모두 삭제한 뒤 새 액션을 추가하면 해당 액션은 `root = true`다.
- [x] 건물/캐릭터/아이템 경로 모두 동일 규칙으로 동작한다.

## 테스트
- [x] `pnpm test:unit` 기준으로 관련 도메인 테스트를 추가/갱신한다.
- [x] 상호작용 타입(건물/캐릭터/아이템)별 생성 테스트를 분리해 검증한다.
- [x] `root` 유일성(최초 1개만 true) 테스트를 추가한다.
- [x] 액션 전부 삭제 후 재추가 시 `root = true` 재부여 테스트를 추가한다.
